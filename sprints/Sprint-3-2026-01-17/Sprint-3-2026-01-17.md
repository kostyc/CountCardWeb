# Sprint 3: Core Infrastructure & Security

**Sprint Date**: January 17, 2026  
**Phase**: Phase 1 - Foundation & Setup  
**Status**: ✅ Completed (Core Functionality) - Testing Pending

## Sprint Goal
Implement core security infrastructure including encryption service, key management, rate limiting, CORS, security headers, input validation, error monitoring, and browser compatibility layer to establish a secure foundation for the application.

## Overview
This sprint focuses on building the security and infrastructure foundation for the CountCard web application. It includes end-to-end encryption with sodium-plus, comprehensive key management, rate limiting middleware, CORS and security headers configuration, Zod validation schemas, Google Cloud Error Reporting integration, and enhanced browser compatibility support.

## Dependencies
- Sprint 1 must be completed (Firebase configuration, project structure, error handling foundation)
- Sprint 2 authentication system should be in progress (for user context in encryption)
- Secure logging utility exists (from Sprint 1) - may need enhancements

## Technical Requirements

### Encryption
- **Library**: sodium-plus
- **Algorithm**: XChaCha20-Poly1305
- **Key Size**: 256 bits (32 bytes)
- **Nonce Size**: 192 bits (24 bytes)
- **Storage Format**: Base64-encoded strings in Firestore

### Rate Limiting
- **Limits**: 20 requests/minute, 100/hour, 500/day
- **Admin Exemption**: Admins exempt from rate limiting
- **Storage**: Upstash Redis or Firestore (serverless-friendly)
- **Headers**: Rate limit headers in all responses

### CORS
- **Allowed Origins**: Configured via `ALLOWED_ORIGINS` environment variable
- **Credentials**: Supported (cookies, authorization headers)
- **Production**: No wildcards allowed

### Security Headers
- **CSP**: Content Security Policy configured
- **X-Frame-Options**: DENY
- **X-Content-Type-Options**: nosniff
- **Referrer-Policy**: strict-origin-when-cross-origin
- **HSTS**: Strict-Transport-Security (if applicable)

### Validation
- **Library**: Zod
- **Coverage**: All API inputs, form data, user inputs
- **Error Format**: User-friendly error messages

### Error Monitoring
- **Service**: Google Cloud Error Reporting
- **Project**: `countcard-94c5b`
- **Sanitization**: No PII in error reports

### Browser Compatibility
- **Target Browsers**: Safari 14+, Chrome 90+, Edge 90+
- **Polyfills**: Loaded conditionally based on feature detection
- **Testing**: Required on all target browsers
