## Sprint Goal
Implement comprehensive recruit management features including recruit profile creation/editing, list view with filtering and search, detail view, USMC rank display system, organizational assignment interface, status management, photo upload, data export, and role-based access control.

## Overview
This sprint focuses on building the complete recruit management system for the CountCard application. It includes creating user interfaces for recruit profile management, implementing filtering and search capabilities, building a USMC rank display system, creating organizational assignment interfaces, implementing role-based access control, and adding GDPR-compliant data export functionality.

## Available Services & Schemas (From Sprint 5)

### Recruit Service (`lib/services/firestore/recruits.ts`)
All recruit data operations are available through the recruit service:

**Core Operations**:
- `createRecruitProfile(recruitId, data, createdBy)` - Create new recruit profile
- `updateRecruitProfile(recruitId, data, updatedBy)` - Update existing recruit profile
- `deleteRecruitProfile(recruitId)` - Delete recruit profile (GDPR compliant)
- `getRecruitProfileById(recruitId)` - Get single recruit by ID

**Query Operations**:
- `listRecruits(filters?, pagination?)` - List recruits with optional filters and pagination
  - Filters: `regiment`, `battalion`, `company`, `series`, `platoon`, `status`, `rank`
- `searchRecruits(searchTerm, pagination?)` - Search recruits by name (firstName/lastName)
- `getRecruitsByOrganization(organization, pagination?)` - Get recruits by organizational unit
- `getRecruitsByStatus(status, pagination?)` - Get recruits by status
- `getRecruitsByRank(rank, pagination?)` - Get recruits by rank
- `getRecruitsByPlatoon(platoon, pagination?)` - Get recruits by platoon

### Validation Schemas (`lib/validation/recruitSchemas.ts`)
All validation schemas are available for form validation:

**Schemas**:
- `recruitCreateSchema` - Validation for recruit creation (Zod schema)
- `recruitUpdateSchema` - Validation for recruit updates (Zod schema)
- `recruitQuerySchema` - Validation for recruit queries/filters (Zod schema)
- `recruitStatusSchema` - Recruit status enum validation
- `platoonSchema` - Platoon format validation (4-digit string: `^\d{4}$`)
- `recruitNameSchema` - Name validation (letters, spaces, hyphens, apostrophes only)

**Type Exports**:
- `RecruitCreateInput` - TypeScript type for recruit creation
- `RecruitUpdateInput` - TypeScript type for recruit updates
- `RecruitQueryInput` - TypeScript type for recruit queries
- `RecruitStatus` - TypeScript type for recruit status
