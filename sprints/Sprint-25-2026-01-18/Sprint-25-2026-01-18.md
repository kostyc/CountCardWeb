## Testing Workflow

**All test results must be documented in [TEST-TRACKING.md](../TEST-TRACKING.md).**

### Step-by-Step Testing Order

Tests should be executed in dependency order, starting with foundational sprints and progressing to feature sprints:

1. **Sprint 1 Testing** (Foundation)
   - Manual steps verification (Firestore database, rules, indexes)
   - Browser compatibility (Safari, Chrome, Edge)
   - Basic functionality (Next.js app, Firebase connection, Tailwind CSS)
   - Security (PII masking, error handling)

2. **Sprint 2 Testing** (Authentication)
   - Authentication flows (email/password, OAuth, phone)
   - Profile creation
   - Role-based access control
   - Password reset flow
   - Privacy policy and terms acceptance

3. **Sprint 3 Testing** (Security)
   - Unit tests for encryption functions
   - Encryption/decryption browser compatibility
   - Security features (rate limiting, CORS, security headers)
   - Integration with Firestore

4. **Sprint 4 Testing** (UI/UX)
   - Component testing (all UI components)
   - Accessibility (WCAG 2.1 AA compliance)
   - Responsive design (mobile, tablet, desktop)
   - Design system verification

5. **Sprint 5 Testing** (Data Models)
   - Type definition validation
   - Validation schema testing (Zod)
   - Service layer testing (CRUD operations)
   - Profile wizard testing
   - Walkthrough system testing

6. **Sprint 6 Testing** (Recruit Management)
   - Recruit CRUD operations
   - List view with filtering and search
   - Role-based access control
   - GDPR compliance (data deletion)
   - Performance testing

7. **Sprint 7 Testing** (Count Cards)
   - Count card workflow (all state transitions)
   - Workflow actions (approve, reject, consolidate, final approve)
   - Role-based access control
   - Performance testing
   - **Note**: Tasks 8 & 9 (audit trail and reporting) are deferred - do not test

8. **Cross-Sprint Testing**
   - End-to-end flows (complete user journeys)
   - Integration testing (data flow across layers)
   - Performance testing (realistic data volumes)
   - Accessibility testing (all features)
   - Browser compatibility testing (all features)
