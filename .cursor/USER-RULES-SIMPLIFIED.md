# Simplified User Rules (Alternative)

If you prefer to keep engineering workflow in User Rules but want a less prescriptive version:

```markdown
# User Rules (Global - applies to all projects)

## Sprint Management
Any sprint that is created will have its own folder to store the sprint itself and all relevant .md files, scripts, etc.

## Engineering Approach
- Plan before coding for complex tasks (use Plan Mode for multi-file changes)
- Make minimal, focused changes that directly address the task
- Review changes for correctness and side effects before completing
- Summarize what was changed and why when delivering work
```

## Comparison

**Full Version (in Project Rules)**: Detailed 5-step procedure with specific requirements
- ✅ More control and discipline
- ✅ Ensures thoroughness
- ❌ Can slow down simple tasks
- ❌ Very prescriptive

**Simplified Version (in User Rules)**: High-level guidelines
- ✅ Flexible for different task types
- ✅ Less friction for quick changes
- ✅ Still encourages good practices
- ❌ Less specific enforcement
